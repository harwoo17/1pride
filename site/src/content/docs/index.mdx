import { Card, CardGrid, LinkCard } from '@astrojs/starlight/components';

A self-paced data curriculum that takes you from `SELECT *` to a deployed
analytics app, using **every game of the Dan Campbell era** (2021–present)
as the substrate and a football-org hierarchy as the level progression.

**33 lessons · 42 challenges · 5 capstones · ~270K rows of real Lions data.**

## The depth chart

<CardGrid stagger>
	<Card title="Level 1 — Analyst" icon="pencil">
		Pull stats on request. `SELECT`, `WHERE`, `ORDER BY`, basic aggregates.
		Capstone: a 2024 season recap notebook.
	</Card>
	<Card title="Level 2 — Position coach" icon="puzzle">
		Develop a position group. `JOIN`, `GROUP BY`, `HAVING`.
		Capstone: a WR efficiency scouting card.
	</Card>
	<Card title="Level 3 — Head coach" icon="seti:clock">
		Situational decisions. Window functions, CTEs, scipy, plotly.
		Capstone: a 4th-down decision EV analyzer.
	</Card>
	<Card title="Level 4 — General manager" icon="seti:db">
		Roster construction. Schema design, scikit-learn, dbt basics.
		Capstone: a draft-pick value model.
	</Card>
	<Card title="Level 5 — Owner" icon="rocket">
		Franchise strategy. FastAPI + Next.js, end to end.
		Capstone: a live app at `app.1pride.app`.
	</Card>
</CardGrid>

## Why this works

**Stakes you actually care about.** Lions football, Dan Campbell era. Domain
familiarity removes the friction of pretending to care about a fake dataset.

**A clean unlock loop.** Each level has a capstone you ship — you advance
because the artifact is real, not because a week passed.

**A natural seniority arc.** Analyst pulls stats. Position Coach compares
groups. Head Coach makes situational calls. GM builds models. Owner ships
the platform. The org chart is the curriculum.
